Who is this for

Do you use AI agents? You need this.

Wait ‚Äî what's MCP?

MCP (Model Context Protocol) is how AI assistants like Claude, Cursor, and VS Code Copilot connect to external tools ‚Äî your database, GitHub, Slack, file system, browser, and more. If you've ever added an MCP server to your Claude Desktop config or Cursor settings, your AI agent is using MCP.

The problem? Anyone can publish an MCP server. There's no review process. Typosquat packages steal your credentials. Known vulnerabilities allow remote code execution. And your config file might have API keys sitting in plaintext ‚Äî visible to the AI model itself.

MCPShield scans your MCP config file and tells you exactly what's wrong before anything bad happens.

Step-by-step guide

How to use MCPShield

0

Prerequisites

You need Node.js installed on your computer. If you can run node --version in your terminal and see a version number, you're good. If not, install it from nodejs.org (download the LTS version).

1

Find your MCP config file

Your MCP config file is where your AI client stores the list of MCP servers you've connected. Here's where to find it:

Client Config file location
Claude Desktop (Mac) ~/Library/Application Support/Claude/claude_desktop_config.json
Claude Desktop (Windows) %APPDATA%\Claude\claude_desktop_config.json
Claude Desktop (Linux) ~/.config/claude/claude_desktop_config.json
Cursor ~/.cursor/mcp.json
VS Code .vscode/mcp.json (in your project)
Windsurf ~/.windsurf/mcp.json
üí° Tip: Not sure where your config is? Just skip to Step 2 and run MCPShield without the --config flag. It will automatically search all of the above locations and scan any configs it finds.
2

Clone and run MCPShield

Open your Terminal (Mac/Linux) or PowerShell (Windows) and run:

Option A ‚Äî Auto-discover all configs
# Clone the repo
git clone https://github.com/mcpshield/mcpshield.git
cd mcpshield

# Scan ‚Äî auto-discovers configs from Claude, Cursor, VS Code, etc.
node src/index.js scan
Option B ‚Äî Scan a specific config file
# Claude Desktop on Mac
node src/index.js scan --config ~/Library/Application\ Support/Claude/claude_desktop_config.json

# Cursor
node src/index.js scan --config ~/.cursor/mcp.json

# VS Code (from your project directory)
node src/index.js scan --config .vscode/mcp.json
3

Read the results

MCPShield scans every server in your config and reports what it finds. Here's what the severity levels mean:

CRITICAL Malicious packages, RCE vulnerabilities, exposed database passwords. Fix immediately.
HIGH Known CVEs, inline API tokens, disabled sandboxes. Fix before production.
MEDIUM Unverified publishers, missing authentication, HTTP connections.
LOW Single maintainer, missing description. Informational.

Every finding includes a green ‚Ü≥ recommendation telling you exactly how to fix it.

4

Fix the issues

The most common fixes are:

üõë Typosquat detected
Remove the malicious server from your config and replace it with the legitimate package name. e.g., change mcp-servr-github to mcp-server-github
üîë Hardcoded credentials
Replace inline secrets with environment variable references. Instead of "GITHUB_TOKEN": "ghp_abc123..." use "GITHUB_TOKEN": "$GITHUB_TOKEN" and set it in your shell.
‚ö†Ô∏è Known CVE
Check if a patch is available (the report tells you). Update the server to the fixed version, or implement the recommended workaround.
5

Add to CI/CD (optional)

For teams, add MCPShield to your build pipeline so MCP configs are scanned on every pull request:

CI/CD integration
# Output JSON report for your pipeline
node src/index.js scan --config mcp.json --json --output report.json

# Exit codes:
# 0 = clean (no high/critical findings)
# 1 = high-severity findings
# 2 = critical findings (block the deploy!)
